1. SanctOS Overview

SanctOS is a privacy-first operating layer built on Solana: a combination of encrypted messaging, a private wallet, and a decentralized node network that eliminates metadata, central servers, and surveillance vectors.

It’s designed to function as:

A secure messenger

A private Solana wallet & relay system

A metadata-hiding node network

Everything encrypted.
Everything local.
Everything under your control.

2. Identity Architecture
Wallet = Identity

Your Solana wallet is your SanctOS identity.

Public key = username

Used to sign permissions

Used to authenticate sessions

Wallet ≠ Encryption Key

Private messaging is not encrypted using your wallet.

This is a major safety upgrade over typical blockchain messaging.

3. Device Key Architecture

Every device generates its own dedicated Ed25519 encryption keypair, separate from your wallet.

Stored locally

Never uploaded

Never in PDA storage

Never transmitted in plaintext

If you log in on 3 devices, you have 3 separate device keys.

Advantages:

Wallet compromise does not leak past messages

Full kill-switch wipe is possible

Per-device trust domains

4. Encryption Model (ECDH + XSalsa20)
Step-by-step

Device A takes A_private_key

Device B takes B_public_key

Both compute an identical ECDH shared secret

SanctOS uses XSalsa20-Poly1305 to encrypt outbound messages

Only the receiving device can decrypt

Encrypted payload is stored on-chain in PDAs

SanctOS (the protocol) cannot decrypt anything.
RPCs cannot decrypt anything.
Nodes (later) cannot decrypt anything.

Messages are encrypted before they ever leave the device.

5. PDA Messaging Layer (On-Chain Storage)

Every conversation has a Program Derived Address:

Immutable

Serverless

Append-only

Stored content:

{
  encrypted_message,
  sender_address,
  sender_device_pubkey,
  timestamp
}


SanctOS does not use:

Servers

Firebase

Chat backends

Centralized storage

Everything is decentralized & censorship-resistant.

6. Kill-Switch Identity System

The SanctOS kill-switch:

Deletes all device keys

Deletes session wallet

Deletes cached conversations

Deletes local preferences

Permanently prevents decrypting old messages

This creates true burnability:

Even you cannot recover old conversations.
No backups.
No plaintext stored anywhere.

Privacy in the strongest sense.

7. Session Wallet (Auto-Sign Messaging)

To allow single-tap messaging:

SanctOS creates an ephemeral session wallet

User authorizes it once

It signs outgoing messaging transactions

Auto-top-ups ensure continuous posting

No Phantom pop-ups

No friction

This enables:

Real-time chat

Lightning-fast message posting

Smooth UX

Without compromising the user’s main wallet.

8. Group Chats (Phase 2)

Encrypted group conversations use:

A shared group symmetric key

Distributed via encrypted device-to-device delivery

Each device stores the group key locally

Key rotates when members join/leave

Same model as Signal:

Efficient

Secure

Zero metadata

This preserves perfect forward secrecy and group privacy.
